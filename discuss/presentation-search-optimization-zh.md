# Agentscan 搜索优化 - 演讲稿(中文口语化版本)

---

大家好,非常感谢你们一直在推动 ERC-8004 生态的发展。

先说说 Agentscan 的现状吧。现在我们是 8004 协议的浏览器,已经索引了超过 5000 个 agents。这个数字还挺不错的!但是说实话,我们的搜索功能真的挺基础的。用户想快速找到合适的 agent 很难。如果我们希望整个生态真正好用起来,这个问题得解决。

所以我们的想法是:优化搜索功能。目标很简单 - 让用户在 3 秒内找到他们需要的 agent。让整个 8004 生态用起来更顺手。

---

我们看了用户反馈和数据,发现了三个主要问题。

**第一个问题:**搜索太慢了。没有自动补全功能。你得把完整的名字打出来才能找到一个 agent。体验不太好。

**第二个问题:**分类形同虚设。我们有 OASF 的 skills 和 domains 显示出来,但那只是展示而已。你没法用它们来筛选。这有啥用呢?

**第三个问题:**完全没有发现机制。没有推荐,没有分类导航,什么都没有。新用户来了完全不知道从哪儿开始。

真正的核心问题是:我们本来应该是个浏览器对吧?但现在基本上就是个搜索框。我们需要更好的索引、更好的导航 - 完整的体验。

---

我们的解决方案分三个方向:让搜索更智能,让导航更清晰,帮助用户真正发现内容。

---

**第一 - 更智能的搜索。**

你开始打字,马上就有智能建议弹出来。我们会展示匹配的 agents、skills、domains,甚至你的搜索历史。

搜索框是空的时候呢?我们展示热门搜索 - 看看其他人都在找什么。

拿到搜索结果后,我们会把你搜的关键词高亮显示。基本功能,但很重要。

---

**第二 - 让分类真正有用。**

我们有 OASF 分类法对吧?咱们把它变成真正的导航。15 个技能集群,10 个领域分类。每个分类都显示里面有多少个 agents。

然后加上高级筛选。网络、技能、领域、信誉评分、创建时间、状态 - 你想怎么组合都行。哦对了,筛选后的 URL 可以分享,你可以把你筛选的视图直接发给别人。

---

**第三 - 帮助用户发现 agents。**

在首页,我们会设置热门分类的快速入口。比如 NLP、DeFi、游戏、代码相关、计算机视觉 - 那些热门的。

我们给你不同的排序方式:综合排名、热门、最新、最高信誉。

两种视图模式:如果你喜欢视觉化就用卡片视图,想要多选对比就用列表视图。

还有推荐功能。首页有"为你推荐"板块。详情页有"相似 Agents"。帮助用户探索。

---

我们还有一些其他想法。Agent 时间线展示链上历史。网络对比功能给部署在多条链上的 agents 用。API 浏览器给想要集成的开发者用。自定义仪表板给高级用户自己配置。

---

性能目标:自动补全要在 100 毫秒以内。搜索结果 300 毫秒以内。首屏加载 2 秒以内。

我们会分阶段逐步推出这些功能。

---

但是,有个很有意思的发现。

我们意识到一件事。Agentscan 实际上有**两类用户**。人类用户...和 AI Agents。

我刚才说的所有东西?主要都是给人类用的。但你们知道吗 - 我们最近看到 GPT、Claude 这些 AI 系统在需要信息的时候,会用基于关键词的网页抓取。比如 Opus 4.5 在找 agent 信息时,会直接做关键词搜索。

所以我们在考虑把 Agentscan 拆成两条产品线。

**产品线一:面向人类的界面。**

就是我刚才说的所有搜索、导航、推荐功能。视觉化界面,按照人类浏览和探索的方式设计。

**产品线二:面向 Agent 的 API。**

专门为 AI Agents 优化的查询接口。这个很不一样。

让我详细说说面向 Agent 的 API 有什么特别的:

**功能一:查询优化。** AI Agents 不像人类那样浏览。它们用关键词查询。所以我们需要针对这种模式做 SEO 优化。结构化的 meta 标签、JSON-LD schema、专门给 agents 爬取的 sitemap。

**功能二:响应格式。** 我们返回干净的、标准化的 JSON。完整的 8004 元数据 - agent 的技能、领域、端点、信誉摘要、链上证明。AI 需要结构化数据,不是 HTML 让它去解析。

**功能三:链上验证。** 当一个 AI Agent 想要查看另一个 agent 时,它需要快速验证链上状态。我们提供快速验证接口。给我们一个 agent ID 或地址,我们返回最新的链上状态 - 所有者、信誉评分、是否活跃。

**功能四:批量查询。** AI 可能需要一次查询多个 agents 来对比。一个 API 请求,多个 agents,返回标准化的对比数据。

**功能五:智能筛选。** AI 用自然语言描述它需要什么。比如"找到 Base 链上信誉超过 80 的 DeFi agents"。我们把自然语言转换成结构化查询,返回匹配结果。

**功能六:Webhooks 和订阅。** AI Agents 可以订阅符合特定条件的 agents。当有新 agents 注册或现有的更新时收到通知。

---

因为 8004 是链上的,我们会重点关注这些:

**链上证明。** 每个查询结果都包含区块号和交易哈希。AI 可以自己验证。

**多链支持。** 我们清楚地告诉你 agent 部署在哪些链上。每条链的状态可能不一样。

**信誉透明。** 我们不只是返回一个分数。我们给你反馈来源、投票者地址、时间戳。让 AI 自己判断可信度。

**数据新鲜度。** 我们准确告诉你数据有多新鲜。最后同步的区块号是多少。AI 就知道它在处理什么数据了。

---

预期效果:

**对人类用户:**3 秒内找到需要的 agent。降低使用门槛,让生态更容易上手。

**对 AI Agents:**成为它们查询 8004 agents 的首选接口。让整个生态的互操作性更好。

这样一来,Agentscan 就不只是个浏览器了。它会成为整个 8004 生态的 **Agent 发现层**。
